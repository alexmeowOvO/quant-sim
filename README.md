# quant-sim

An event-driven backtester and daily signal monitor for ASX mean-reversion strategies, running on 1-hour bars via Yahoo Finance.

## Strategy

Mean reversion for ranging (non-trending) stocks. Entries require four conditions to hold simultaneously:

- **Bollinger Band touch** — close at or below the lower band
- **VWAP discount** — price trading below VWAP by a minimum threshold
- **Low ADX** — ADX below tier threshold (ranging regime only; trending stocks are skipped)
- **Volume climax** — high-volume bar with a small body signals selling exhaustion

Market filter: no new entries when the ASX 200 (`^AXJO`) is below its 50-bar SMA.

Exit on BB midline / VWAP signal reversal, ATR-based trailing stop, or hold timeout.

## Volatility tiers

Stocks are split into two tiers with independent signal and risk parameters, assigned by their historical ranging behaviour:

| Tier | Criteria | BB | ATR stop | Max hold |
|------|----------|----|----------|----------|
| tier_a | ranging% > 25% | 20-bar / 1.0σ | 3.5× ATR | 18 bars |
| tier_b | ranging% ≤ 25% | 10-bar / 1.5σ | 4.5× ATR | 16 bars |

Parameters were selected by per-tier sweep over 730 days of 1h data, optimising Sharpe ratio independently for each tier.

## Universe

73 ASX stocks across tier_a (42) and tier_b (31), screened for:

- `ranging_pct` ≥ 25% (tier_a) or ≥ 15% (tier_b) — stock must actually range
- `vwap_dev_freq` ≥ 15% — price must deviate from VWAP often enough to fire entries
- ATR between 0.45 and 1.00 — not too tight, not too volatile
- ≥ 3000 bars of clean 1h data over the backtest window

Stocks failing these thresholds are logged to `results/deleted_stocks.csv`. Universe composition is driven by behaviour, not sector diversity.

## Usage

```bash
pip install -r requirements.txt

# Full backtest (writes results/ artifacts)
python3 main.py

# Backtest against a specific config
python3 main.py --config config/baseline_73stocks_sharpe1975.yaml

# Per-tier parameter sweep (saves sweep_tier_a.csv / sweep_tier_b.csv)
bash run_all_sweeps.sh

# Daily signal check (normally run by cron at 5pm Sydney time)
python3 signal_logger.py

# Screen new candidate stocks against behaviour thresholds
python3 measure_behaviour.py
python3 auto_rotate.py
```

## Daily signal logger

`signal_logger.py` runs every weekday at 5pm Sydney time via cron. It checks the full 73-stock universe for live entry signals and near-misses, and appends to:

- `results/daily_signals.csv` — machine-readable log
- `results/daily_signals.log` — human-readable daily summary

Near-miss threshold: price within 0.5% of the BB lower band. Useful for spotting stocks approaching an entry before the next session.

Cron entry (set once with `crontab -e`):
```
TZ=Australia/Sydney
0 17 * * 1-5 /usr/bin/python3 /path/to/quant-sim/signal_logger.py >> /path/to/quant-sim/results/signal_logger_cron.log 2>&1
```

## Engine notes

**Mark-to-market** uses the unified timestamp timeline across all stocks. When a stock has no bar at a given timestamp (Yahoo Finance data gaps, cross-listed calendar differences), price is forward-filled from the last known close rather than zeroed. This prevents spurious equity spikes from inflating max drawdown. Forward-fill events are logged to `results/missing_bar_log.csv`.

**Entry execution** fills at the next bar's open after the signal bar, with slippage applied. Exit execution fills at the current bar's open when the previous bar's exit signal is True (decision and execution on separate bars).

**Position sizing** is 10% of current portfolio value per trade, max 8 concurrent positions.
